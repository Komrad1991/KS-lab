# Спецификация: Командная Палитра (Command Palette) для TiniJS

## Цель

Добавить полнофункциональную систему палитры команд с поддержкой нечеткого поиска, горячих клавиш, контекстной фильтрации, навигации по роутерам, аргументов команд, асинхронности, отмены и темизации.

## 1. Базовый компонент CommandPalette (пакет @tinijs/ui)

- Открытие по `Cmd/Ctrl+K` (глобальный хоткей)
- Оверлей/модальное окно с палитрой команд
- Поле поиска с нечетким поиском (fuzzy search) и подсветкой совпадений
- Список доступных команд: иконка, название, описание, горячие клавиши
- Навигация по списку стрелками вверх/вниз
- Активация выбранной команды по Enter
- Закрытие по Esc и клику вне окна
- Поддержка аргументов команд:
  - Если у команды есть `args`, после выбора показывается форма ввода/селекта для каждого аргумента
  - Кнопка «Запустить» выполняет команду с заполненными аргументами
  - Отмена аргументного ввода через ссылку «Отмена»
- Индикация загрузки: спиннер (или другой индикатор) рядом с командой во время выполнения
- Отмена выполнения: нажатие Esc во время выполнения асинхронной команды отправляет `AbortSignal` в `action` и останавливает задачу
- Визуальное отображение зарегистрированных горячих клавиш рядом с командами (компонент `tini-shortcut`)
- Поддержка тем: светлая/темная; стилизация под 9 тем оформления (Bootstrap, Material, iOS, Fluent, Radix, Chakra, Daisy, Shadcn + кастомная Tini)
- История команд: хранение в localStorage; ранжирование по частоте/свежести использования

## 2. Архитектура и API

### Компоненты UI

- `tini-command-palette` — основное модальное окно (Web Component, расширяет `TiniComponent`)
- `tini-command-item` — элемент списка команды
- `tini-shortcut` — отображение горячих клавиш

### CommandService (@tinijs/toolbox)

- Типы:
  - `CommandArgument` (name, type, options?, description?)
  - `CommandDefinition` (id, name, description?, icon?, action: (args?, abortSignal?) => Promise<void>|void, keywords?, category?, shouldShow?, args?)
  - `ShortcutDefinition` (keys, commandId, preventDefault?)
- API:
  - `registerCommand(...)`
  - `unregisterCommand(id)`
  - `getRegisteredCommands()`
  - `getCommandsByCategory()`
  - `searchCommands(query, context?)` — fuzzy + фильтр by shouldShow + history boost
  - `runCommand(id, args?, abortSignal?)` — поддерживает AbortSignal
  - `getShortcut(commandId)` — возвращает последний зарегистрированный хоткей
  - `subscribeContext(cb)`, `getContext()`, `updateContext(ctx)`
  - `getHistory()`, `getRecentCommands(limit?)`

### Горячие ключи (HotkeyManager)

- `registerShortcut(keys, commandId, preventDefault?)`
  - Поддержка одиночных комбинаций (`Ctrl+K`) и последовательностей (`g h`)
  - При конфликте: последний зарегистрированный имеет приоритет (last-write-wins)
  - Последовательности: таймаут 800 мс между шагами
- `unregisterShortcut(keys, commandId?)`

### Интеграция с роутером

- `autoRegisterRoutesFromRouter(service, router)` — автоматически регистрирует навигационные команды для всех маршрутов
  - Использует `router.routes` или `router.getRoutes()`
  - Формат команды: `"Перейти к [Название]"` (или на основе `route.title`)
  - Навигация через `router.go(path)` или `history.pushState`
- `registerNavigationCommandsFromRoutes(service, routes, navigateFn?)` — низкоуровневая функция

### История и ранжирование

- Хранение: `localStorage` ключ `tinijs.history.commands` (массив ID)
- При выполнении команды (`runCommand`) ID добавляется в историю (limit 50)
- В `searchCommands` команды из истории получают бонус +1000 к fuzzy-скорингу, чтобы оказываться выше
- Дополнительно учитывается частота использования, чтобы популярные команды поднимались выше среди релевантных результатов
- Навигационные команды получают recent-статус не только после запуска из палитры, но и после обычных переходов роутера

## 3. Функциональные требования

### Базовые

- Создание компонента `TiniCommandPalette` в `@tinijs/ui`
- Открытие по Cmd/Ctrl+K, закрытие по Esc и клику вне окна
- Поле поиска (ввод в реальном времени), fuzzy-фильтрация
- Навигация стрелками, Enter — выполнение
- Отображение списка команд: иконка, название, описание, shortcut
- Горячие клавиши: регистрация, отображение, конфликт-разрешение (последний win)
- Последовательности клавиш (например, "g h")
- Группировка команд по категориям (`category`)

### Контекст и условие

- `shouldShow(context)` для каждой команды — позволяет показывать команды только в определенных страницах/состояниях
- `subscribeContext` и `updateContext` — обновление текущего контекста
- Команды могут динамически регистрироваться/удаляться в рантайме

### Роутер

- Автоматическая регистрация навигационных команд из `router.routes`
- Формат: "Перейти к [Название страницы]"
- Быстрый доступ к недавно посещенным страницам через команды навигации (учитываются в истории)

### Аргументы

- Команда может объявить `args: CommandArgument[]`
- При выборе такой команды в палитре показывается интерактивная форма (input/select)
- После заполнения и нажатия «Запустить» команда выполняется с этими аргументами

### Асинхронность и отмена

- `action` может возвращать `Promise`
- Палитра показывает индикатор загрузки (спиннер) для выполняемой команды
- Нажатие `Esc` во время выполнения асинхронной команды отправляет `AbortSignal` в `action`
- Поддержка отмены через `AbortController` внутри `CommandService.runCommand`

### Темы

- Поддержка светлой/темной темы через свойство `theme` на компоненте (`'light'|'dark'`)
- Стили используют CSS-переменные (`--cp-bg`, `--cp-fg`, ...) для переключения
- Легкость добавления тем для 9 UI-фреймворков (Bootstrap, Material, iOS, Fluent, Radix, Chakra, Daisy, Shadcn + кастомная Tini)

## 4. Тестирование

- Unit-тесты `CommandService`:
  - регистрация/удаление команд

- Unit-тесты `HotkeyManager`:
  - одиночные комбинации
  - последовательности (таймаут, сброс)
  - конфликты (последний перезаписывает)
- Интеграционные тесты:
  - открытие палитры, поиск, выбор, выполнение
  - роутер-интеграция (навигационные команды)
  - аргументы (ввод и выполнение)
  - отмена асинхронной команды

## 6. Ожидаемый результат (Acceptance Criteria)

- Палитра открывается Cmd/Ctrl+K
- Поиск работает (fuzzy), результаты ранжируются (fuzzy + история)
- Навигация стрелками, Enter выполняет команду
- Закрытие Esc и кликом вне окна
- Hotkeys работают (последний перезаписывает); конфликт-решение
- Роутер-интеграция: навигационные команды автоматически зарегистрированы, формат "Перейти к [Название]"
- История команд хранится и используется для подсказок
- Поддержка светлой/темной темы и 9 тем оформления
- Асинхронные команды и отмена выполнения; индикаторы загрузки
- Пример использования в apps/demo-command-palette присутствует
- Unit-тесты покрытия ≥80%

## 7. Примечания

- Реализация последовательностей клавиш использует простой state machine с таймаутом 800 мс
- Приоритет хоткеев: последний зарегистрированный (можно переопределить при необходимости)
- Темы: CSS-переменные позволяют легко добавлять новые темы; в будущем можно создать набор тем для популярных UI-фреймворков
- История хранится в localStorage; в SSR/небраузерных средах отключается
- fuzzy-поиск без transliteration (можно расширить)
