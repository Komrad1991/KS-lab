# ТЗ: Командная Палитра (Command Palette) для TiniJS

Цель: добавить базовую систему палитры команд, доступную через Cmd/Ctrl+K, с поиском, навигацией, выполнением и историей.

1. Базовый компонент CommandPalette (
пакет @tinijs/ui)
- Открытие по сочетанию Cmd/Ctrl+K.
- Оверлей-модальное окно с палитрой команд.
- Поле поиска с поддержкой нечеткого поиска (fuzzy search).
- Отображение доступных команд: иконка, название, описание.
- Навигация по списку стрелками вверх/вниз.
- Активирование выбранной команды по Enter.
- Закрытие по Esc и клику вне окна.
- Возможность полнотекстового фильтра и подсветки совпадений.
- Визуальное отображение зарегистрированных клавиатурных сокращений рядом с командами.
- Поддержка аргументов команды и параметров вызова (примеры в API).
- Сохранение истории использования в localStorage.
- Поддержка тем (светлая/темная) и стилизация под 9 тем оформления.

2. Архитектура и API
- Компонент: tini-command-palette, tini-command-item, tini-shortcut.
- API для регистрации команд будет вынесен в CommandService, пакет @tinijs/toolbox.
- Источники данных: список команд, контекст страницы, история, список роутов.
- Псевдонимы и keywords для fuzzy-поиска: по каждому команде хранить keywords.

3. Функциональные требования
- Регистрация команд через CommandService: registerCommand(id, name, description, icon, action, keywords?).
- Группировка команд по категориям; поддержка динамической регистрации/удаления.
- Условия отображения команд по контексту/странице.
- Регистрация горячих клавиш через registerShortcut(keys, commandId, preventDefault?).
- Поддержка модальных последовательностей клавиш (например, "g h").
- Разрешение конфликтов сочетаний клавиш: при конфликте использовать более приоритетную команду или показывать уведомление.
- Автопополнение навигации по ролям маршрутов через @tinijs/router: команды типа "Перейти к [Название]".
- Быстрый доступ к недавно посещенным страницам.
- Асинхронные команды с индикатором загрузки, отмена выполнения и обработка ошибок.

4. Хранилище и история
- История команд сохраняется в localStorage (ключ: tinijs.history.commands).
- История используется для ранжирования и подсказок.
- Реализация сортировки по релевантности и частоте использования.

5. Тестирование и документация
- unit-тесты для CommandService и механики регистрации команд.
- тесты для алгоритма fuzzy search.
- тесты по обработке клавиатурных событий.
- пример использования в apps/ (пример-прикладное приложение).
- документация API и примеры использования.

6. Технические детали
- Язык: TypeScript.
- Базовый класс: TiniComponent из @tinijs/core.
- Использовать Input, Dialog, Spinner из @tinijs/ui.
- Хранилище состояния истории: createStore из @tinijs/store.
- Для fuzzy search можно использовать transliterate или реализовать свой алгоритм.
- Экспорты в public-api.ts соответствующих пакетов.

7. Ожидаемый результат
- Открытие палитры Cmd/Ctrl+K, поиск, навигация, Enter — выполнение.
- Работают зарегистрированные сокращения клавиш.
- Видна история недавно использованных команд.
- Возможность перехода к страницам приложения через палитру.

Примечание: эти файлы служат техническим заданием и не требуют изменений в коде на текущем этапе.

Public API и документацию планируется описать отдельно после согласования архитектуры.
