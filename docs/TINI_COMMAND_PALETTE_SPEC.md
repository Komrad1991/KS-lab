# Спецификация: Командная Палитра (Command Palette) для TiniJS

Цель задачи
- Добавить базовую систему палитры команд с поддержкой поиска, горячих клавиш и контекстной навигации в рамках TiniJS.

1. Функциональные требования
- Создать компонент CommandPalette в пакете @tinijs/ui.
- Палитра открывается по сочетанию Cmd/Ctrl+K; реализован оверлей с модальным окном.
- Поле поиска с нечетким поиском (fuzzy search); подсветка совпадений.
- Отображение списка команд: иконка, название, описание, клавиатуры (hotkeys).
- Навигация в списке стрелками вверх/вниз; активация Enter; закрытие Esc.
- Закрытие палитры кликом вне окна.
- Поддержка фильтрации по контексту и категорий; сохранение истории использования в localStorage.
- Адаптация под 9 UI-тем (Bootstrap, Material, iOS, Fluent, Radix, Chakra, Daisy, Shadcn, и др.) и поддержка светлой/темной темы.
- Поддержка аргументов команд и асинхронных вызовов (индикатор загрузки, отмена).

2. Архитектура и API
- UI-компоненты: tini-command-palette, tini-command-item, tini-shortcut.
- Логика команд вынесена в CommandService (@tinijs/toolbox): регистрация, группировка, контекст, навигация, горячие клавиши.
- Источник данных: зарегистрированные команды, контекст, история, маршруты.
- Поддержка keywords для нечеткого поиска.
- Поддержка динамической регистрации/удаления команд; shouldShow(context) для контекстной фильтрации.
- Регистрация горячих клавиш: registerShortcut(keys, commandId, preventDefault?). Поддержка последовательностей и модификаторов.
- Конфликты хоткеев: при конфликте — приоритетная команда или уведомление.
- Интеграция с роутером: автоматическая регистрация навигационных команд по роутам; формат: "Перейти к [Название]".
- История команд: хранение в localStorage (tinijs.history.commands); ранжирование по релевантности и частоте использования.
- Поддержка аргументов команд: auto-completion и передача args в action.

3. Типы и API (пример)
- CommandDefinition: { id: string; name: string; description?: string; icon?: string; action: (args?: any) => Promise<void> | void; keywords?: string[]; category?: string; }
- ShortcutDefinition: { keys: string; commandId: string; preventDefault?: boolean }
- Методы CommandService: registerCommand, unregisterCommand, getRegisteredCommands, registerShortcut, unregisterShortcut, subscribeContext, getContext, etc.

4. Взаимодействие с UI
- Палитра использует: tini-command-palette, tini-command-item, tini-shortcut.
- Команды отображаются с иконками и описаниями; рядом — визуальная подсветка клавиатурных сокращений.
- Поиск поддерживает fuzzy-search; можно использовать transliterate как опцию.

5. Тестирование и документация
- unit-тесты для CommandService, регистрации/удаления, группировок, shouldShow.
- тесты fuzzy search и обработки клавиатурных событий.
- интеграционные тесты для навигации и регистрации роутеров.
- пример_usage в apps/ и документация по API.

6. Технические детали
- Язык: TypeScript.
- Базовый класс: TiniComponent из @tinijs/core.
- Использовать Input, Dialog, Spinner из @tinijs/ui.
- Хранилище истории команд: createStore из @tinijs/store.
- Флуфу-сервис: fuzzy search; transliterate предпочтительно как опция.
- Экспорт в public-api.ts соответствующих пакетов.

7. Ожидаемые результаты
- Открытие палитры Cmd/Ctrl+K; ввод запроса; навигация; Enter для выполнения.
- Регистрация и использование клавиатурных сокращений.
- История недавно использованных команд.
- Переход к страницам приложения через палитру.

8. Примечания
- Детали реализации и архитектура будут уточнены на этапе прототипирования.
