# Условия работы над задачей

1. Стек и базовые конвенции
- Язык: TypeScript.
- Базовые классы: TiniComponent из @tinijs/core.
- UI-компоненты: используем @tinijs/ui (Input, Dialog, Spinner).
- Хранилище состояния: createStore из @tinijs/store.
- Нормы экспорта: public-api.ts должны раскрывать новые модули/сервисы.

2. Правила разработки и тестирования
- Не изменять существующий код на данном этапе; сначала подготовить ТЗ и архитектуру.
- При переходе к реализации — писать unit-тесты для новых модулей.
- Тестировать fuzzy search отдельно, как отдельный модуль.
- Примеры использования — в apps/ (один приложение-пример).

3. Архитектурные принципы
- Разделение: UI-поведение (палитра) отдельно от бизнес-логики регистрации команд (CommandService).
- Контекстные команды должны зависеть от состояния приложения и текущего маршрута.
- Асинхронность: поддерживать асинхронность команд с индикаторами загрузки.

4. Интеграция с роутером
- Автоматическая регистрация навигационных команд для роутов, чтобы обеспечивать быстрый переход между страницами.
- Команды навигации должны формироваться как "Перейти к [Название]".

5. Поддержка тем и стилей
- Палитра и элементы должны поддерживать светлую и тёмную темы; стиль должен интегрироваться с 9 стилевыми темами.
- Использовать существующие средства стилизации UI-пакета.

6. Безопасность и устойчивость
- Обращение к localStorage для истории команд; обработка ошибок доступа к локальному хранилищу.
- Обработка отмены выполнения команд.
- Возможность отката действий, если команда не удовлетворяет контексту.

7. Документация
- API-справка и примеры использования в отдельных разделах.
- Комментарии в патчах и архитектурных файлах.

Примечание: изменения не вносятся в код до утверждения спецификации.
